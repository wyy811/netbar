### 2.4 中间件

#### 自定义中间件
```go
// 日志中间件
func LoggerMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        
        // 请求前
        c.Next() // 执行后续处理
        
        // 请求后
        duration := time.Since(start)
        log.Printf("%s %s %v", c.Request.Method, c.Request.URL.Path, duration)
    }
}

// 使用
r.Use(LoggerMiddleware())
```

#### 中间件顺序

### 2.5 参数校验

```go
// 结构体绑定 + 校验
type LoginRequest struct {
    Username string `json:"username" binding:"required,min=3,max=20"`
    Password string `json:"password" binding:"required,min=6"`
}

// 在 Handler 中使用
func Login(c *gin.Context) {
    var req LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }
    // 继续业务
}
```

#### 常用校验标签
| 标签       | 说明     | 示例                    |
| ---------- | -------- | ----------------------- |
| `required` | 必须     | `binding:"required"`    |
| `min`      | 最小值   | `binding:"min=1"`       |
| `max`      | 最大值   | `binding:"max=100"`     |
| `email`    | 邮箱格式 | `binding:"email"`       |
| `oneof`    | 枚举     | `binding:"oneof=1 2 3"` |
| `gt`       | 大于     | `binding:"gt=0"`        |
| `url`      | URL 格式 | `binding:"url"`         |

## 第三部分：GORM 核心（🎯 必学）

### 3.1 模型定义

```go
import "gorm.io/gorm"

type User struct {
    gorm.Model                   // 内嵌 ID, CreatedAt, UpdatedAt, DeletedAt
    Username    string `gorm:"uniqueIndex;size:50;not null"`
    Password    string `gorm:"column:password_hash;size:255"`
    Age         int    `gorm:"default:0"`
    Email       string `gorm:"size:100"`
}

// 自定义表名
func (User) TableName() string {
    return "users"
}
```

#### 常用标签
| 标签          | 说明       |
| ------------- | ---------- |
| `column`      | 指定列名   |
| `type`        | 列类型     |
| `size`        | 长度       |
| `primaryKey`  | 主键       |
| `uniqueIndex` | 唯一索引   |
| `index`       | 普通索引   |
| `default`     | 默认值     |
| `not null`    | 非空       |
| `-`           | 忽略该字段 |

### 3.2 CRUD

#### 创建
```go
user := User{Username: "admin", Age: 25}
result := db.Create(&user)
if result.Error != nil {
    // 处理错误
}
fmt.Println(user.ID) // 获取自增 ID
```

#### 查询
```go
// 查询单条
var user User
db.First(&user, 1)              // 根据主键
db.First(&user, "username = ?", "admin")  // 条件
db.Last(&user)

// 查询多条
var users []User
db.Find(&users)                 // 所有
db.Where("age > ?", 18).Find(&users)
db.Where("username LIKE ?", "%张%").Find(&users)

// 条件组合
db.Where("age > ?", 18).Where("status = ?", 1).Find(&users)

// 排序/分页
db.Order("age desc").Limit(10).Offset(20).Find(&users)

// 选择特定字段
db.Select("id", "username").Find(&users)

// 统计
var count int64
db.Model(&User{}).Where("age > ?", 18).Count(&count)
```

#### 更新
```go
// 更新单个字段
db.Model(&user).Update("age", 26)

// 更新多个字段
db.Model(&user).Updates(User{Age: 26, Email: "new@email.com"})
db.Model(&user).Updates(map[string]interface{}{
    "age": 26,
    "email": "new@email.com",
})

// 更新指定字段（排除零值）
db.Model(&user).Select("age").Updates(User{Age: 0}) // 会更新为0
```

#### 删除
```go
// 软删除（有 gorm.Model 时自动使用）
db.Delete(&user)  // 设置 deleted_at，不会真正删除

// 硬删除
db.Unscoped().Delete(&user)
```

### 3.4 事务（🎯 重要）

```go
// 自动事务
err := db.Transaction(func(tx *gorm.DB) error {
    // 操作1：创建订单
    if err := tx.Create(&order).Error; err != nil {
        return err  // 回滚
    }
    
    // 操作2：更新库存
    if err := tx.Model(&product).Update("stock", gorm.Expr("stock - ?", 1)).Error; err != nil {
        return err  // 回滚
    }
    
    return nil  // 提交
})

// 手动事务
tx := db.Begin()
// ... 操作 ...
if err != nil {
    tx.Rollback()
} else {
    tx.Commit()
}
```

### 4.4 响应式 API（🎯 核心）

#### ref
```javascript
import { ref } from 'vue'

const count = ref(0)
const user = ref(null)

// 读写
count.value = 10
console.log(count.value)
```

#### reactive（仅对象）
```javascript
import { reactive } from 'vue'

const state = reactive({
    count: 0,
    user: { name: '张三' }
})

state.count = 10
state.user.name = '李四'
```

**📌 重要区别**：
- `ref`：任意类型，访问需 `.value`
- `reactive`：仅对象，直接访问属性
- `reactive` 对象不能解构，解构会失去响应性

#### toRefs（解构 reactive）
```javascript
const state = reactive({ count: 0, name: '张三' })

// 解构后依然保持响应
const { count, name } = toRefs(state)
// 使用：count.value
```

### 4.5 计算属性和侦听器

#### computed
```javascript
import { ref, computed } from 'vue'

const firstName = ref('张')
const lastName = ref('三')

// 只读
const fullName = computed(() => {
    return firstName.value + lastName.value
})

// 可写
const fullName2 = computed({
    get: () => firstName.value + lastName.value,
    set: (val) => {
        firstName.value = val[0]
        lastName.value = val[1]
    }
})
```

#### watch / watchEffect
```javascript
import { watch, watchEffect } from 'vue'

// 监听单个
watch(count, (newVal, oldVal) => {
    console.log('变化:', newVal, oldVal)
})

// 监听多个
watch([count, name], ([newCount, newName]) => {
    console.log('count 或 name 变化了')
})

// 立即执行 + 深度监听
watch(state, (newVal) => {
    console.log('state 变了')
}, { deep: true, immediate: true })

// 自动追踪依赖
watchEffect(() => {
    console.log('count 或 name 变化:', count.value, name.value)
})
```

## 第六部分：Pinia 状态管理（🎯 必学）

### 6.1 Store 定义

```javascript
// stores/auth.store.js
import { defineStore } from 'pinia'

export const useAuthStore = defineStore('auth', () => {
    // State
    const token = ref(null)
    const user = ref(null)
    
    // Getters
    const isLoggedIn = computed(() => !!token.value)
    const username = computed(() => user.value?.name || '')
    
    // Actions
    function setAuth(data) {
        token.value = data.token
        user.value = data.user
        localStorage.setItem('token', data.token)
    }
    
    function logout() {
        token.value = null
        user.value = null
        localStorage.removeItem('token')
    }
    
    return {
        token,
        user,
        isLoggedIn,
        username,
        setAuth,
        logout
    }
})
```

## 第七部分：Axios（🎯 必学）

### 7.1 基础配置

```javascript
// api/client.js
import axios from 'axios'

const client = axios.create({
    baseURL: import.meta.env.VITE_API_BASE_URL,
    timeout: 10000,
    headers: {
        'Content-Type': 'application/json'
    }
})

// 请求拦截器
client.interceptors.request.use(
    config => {
        const token = localStorage.getItem('token')
        if (token) {
            config.headers.Authorization = `Bearer ${token}`
        }
        return config
    },
    error => Promise.reject(error)
)

// 响应拦截器
client.interceptors.response.use(
    response => response.data,  // 直接返回 data
    error => {
        if (error.response?.status === 401) {
            // Token 过期，跳转登录
            localStorage.removeItem('token')
            window.location.href = '/login'
        }
        return Promise.reject(error)
    }
)

export default client
```

## 第十部分：常见错误速查

### 后端常见错误

| 错误                           | 原因       | 解决                                |
| ------------------------------ | ---------- | ----------------------------------- |
| `cannot find module`           | 包未下载   | `go mod tidy`                       |
| `undefined: gin`               | 未导入     | `import "github.com/gin-gonic/gin"` |
| `missing go.sum entry`         | 依赖问题   | `go mod download`                   |
| `bind: address already in use` | 端口被占用 | 换端口或结束占用进程                |
| `invalid memory address`       | 空指针     | 检查是否未初始化                    |

### 前端常见错误

| 错误                                 | 原因           | 解决                |
| ------------------------------------ | -------------- | ------------------- |
| `Cannot read property of undefined`  | 访问未定义属性 | 使用 `?.` 或 `v-if` |
| `Reactive variable loses reactivity` | 解构 reactive  | 使用 `toRefs`       |
| `Component not found`                | 组件未注册     | 检查导入和路径      |
| `CORS error`                         | 跨域           | 后端配置 CORS       |
| `Module not found`                   | 包未安装       | `npm install`       |
