# 项目框架核心流程图

基于你提供的文档，我用流程图直观展示这个网吧管理系统的整体架构和各层职责：

---

## 一、整体架构分层图（核心）

```mermaid
flowchart TB
    subgraph 前端["🖥️ 前端层 (Vue3 + Pinia)"]
        V["📄 Views<br>页面组件"]
        C["🧩 Components<br>通用组件"]
        S["📦 Stores<br>状态管理"]
        API["🌐 API<br>请求层"]
    end

    subgraph 后端["⚙️ 后端层 (Go + Gin)"]
        R["🛣️ Router<br>路由注册"]
        M["🛡️ Middleware<br>中间件"]
        H["🎮 Handler<br>控制器"]
        SV["💼 Service<br>业务逻辑"]
        RE["🗄️ Repository<br>数据访问"]
        D["📦 Domain<br>领域模型"]
    end

    subgraph 基础设施["🗃️ 基础设施"]
        DB["MySQL<br>数据库"]
        CACHE["Redis<br>缓存(可选)"]
    end

    V --> C
    V --> S
    S --> API
    API -->|HTTP请求| R
    R --> M --> H
    H --> SV
    SV --> RE
    RE --> DB
    SV --> D
    RE --> D
```
